F1
Function 1

Stream Video from IP Camera

1 mark
1

Connect to Source

Accepts int (webcam index 0) or str RTSP URL (rtsp://192.168.x.x/stream).
Uses CAP_DSHOW for webcam, CAP_FFMPEG for RTSP.

2

Set Buffer Size = 1

Forces OpenCV to keep only the latest frame — prevents stale frame accumulation in the buffer.

3

Background Thread

A daemon thread runs _update() continuously, reading frames with a threading.Lock. The UI thread calls .read() and gets the latest frame instantly from RAM with zero I/O wait.

4

Context Manager Support

Supports with CameraStream() as cam: — auto-disconnects on exit.

class CameraStream: def connect(self, source): backend = cv2.CAP_DSHOW if isinstance( source, int) else cv2.CAP_FFMPEG self._cap = cv2.VideoCapture( source, backend) self._cap.set( cv2.CAP_PROP_BUFFERSIZE, 1) # Start background reader thread self._thread = threading.Thread( target=self._update, daemon=True) self._thread.start() def _update(self): while self._running: ok, frame = self._cap.read() if ok: with self._lock: self._frame = frame def read(self): with self._lock: return self._frame.copy()

Function 2

Crop Video into Image Frames

1 mark
1

Open Video Source

Accepts webcam, RTSP stream, or a .MOV / .mp4 file. For files: extracts 1 frame every FRAME_INTERVAL = 15 frames.

2

Pre-process Frame

① Mirror horizontally (cv2.flip) — corrects webcam reversal.
② Resize to max 640 px wide (INTER_AREA) — speeds up detection.

3

De-duplicate Check

Convert to 100×100 grayscale thumbnail → compute mean absolute diff from previous saved frame. If diff < 8.0, skip — ensures diverse dataset.

4

Save to Dataset

Save up to MAX_IMAGES = 30 JPEGs per student into dataset/{ID}_{Name}/frame_001.jpg. Uses cv2.imencode + tofile() for Unicode path safety.

Pre-processing (Optional +1 mark)

Light Balance & Noise Filtering

+1 mark
🎨

Step 1 — Grayscale

Convert BGR colour frame to grayscale using cv2.cvtColor(BGR2GRAY).

Removes colour information — face detection works on luminance only. Reduces computation by ⅓.

🌤️

Step 2 — CLAHE

Contrast Limited Adaptive Histogram Equalisation

Parameters: clipLimit=2.0, tileGrid=(8,8)

Enhances local contrast in different regions independently → makes faces detectable under poor / uneven lighting without over-brightening.

🌫️

Step 3 — Gaussian Blur

cv2.GaussianBlur(kernel=(3,3), σ=0)

Smooths high-frequency noise (sensor grain, compression artefacts) before the Haar Cascade runs → fewer false positives.

Note: applied only for detection, not saved to dataset.

Pre-processing pipeline applied in FaceDetector.preprocess(frame): BGR Frame → Grayscale → CLAHE → Gaussian Blur → Used for detection only
F3
Function 3

Face Detection — Haar Cascade

2 marks
1

Pre-process Frame

Grayscale + CLAHE + Gaussian Blur (see Pre-processing slide). This improves detection sensitivity under variable lighting.

2

Run detectMultiScale

scaleFactor=1.2 — scale pyramid step (20% shrink per level).
minNeighbors=5 — minimum overlapping detections to confirm.
minSize=(70,70) — ignore faces smaller than 70×70 px.

3

Select Largest Face

If multiple faces detected, pick the one with maximum area (w × h) — guarantees we process the student standing closest to the camera.

4

Crop + Normalise

Crop from the original colour frame → convert to grayscale → resize to IMAGE_SIZE = (112, 112) with INTER_AREA. Returns FaceBox(x,y,w,h) + face image.

Haar Cascade — How It Works

The cascade is a series of boosted classifiers (AdaBoost). Each stage uses Haar-like features — rectangular filters that measure intensity differences (dark eye region vs bright cheek region, etc.).

A sliding window scans the image at multiple scales. If a region passes all stages, a face is declared. Early rejection at each stage makes it very fast.

haarcascade_frontalface_default.xml OpenCV built-in Output: 112×112 grayscale
F4
Function 4

Face Recognition — Centroid Classifier

2 marks

🏋️ Training Phase

1

Vectorize Each Face

Apply equalizeHist → flatten 112×112 → divide by 255 → 12 544-dimensional float32 vector

2

Compute Centroid

For each student: centroid = mean(all vectors). One point in 12 544-D space per student.

3

Auto-calibrate Threshold

Measure own-class vs wrong-class distance distributions. Set threshold = min(16.0, p95_own×1.15, p5_wrong×0.85)

🔍 Inference Phase

1

Vectorize Live Face

Same pipeline: equalizeHist → flatten → /255

2

Euclidean Distance

‖vector − centroid_i‖₂ for every student. Pick argmin.

3

Reject Unknown

If best_dist > threshold → Unknown.
If gap to 2nd-best < 1.2 → ambiguous → Unknown.

4

Log Attendance

Append student_id, name, timestamp to attendance.csv. Each student logged once per session.

Summary

What We Built

📡

Function 1 — RTSP Stream

Threaded camera reader supports both local webcam and IP camera via RTSP protocol. Zero-latency frame delivery to UI thread.

🎞️

Function 2 — Frame Extraction

Video-to-dataset pipeline with smart de-duplication, mirror correction, and configurable sampling rate. Saved 554 images across 6 students.

🌤️

Pre-processing

CLAHE adaptive histogram equalisation + Gaussian blur improve detection under varied classroom lighting conditions — implemented in the detection pre-processing step.

👁️

Function 3 — Haar Cascade Detection

Detects and crops the largest frontal face per frame. Output: 112×112 grayscale image ready for recognition.

🧠

Function 4 — Centroid Recognition

No-neural-network classifier: centroid per student in 12 544-D feature space. Auto-calibrated threshold + ambiguity rejection. Trains in < 1 second.

📋

Attendance Logging

Automatic CSV log with student ID, name, and timestamp. Each student recorded once per session. Desktop UI with register & attendance modes.
